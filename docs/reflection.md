#Reflection

#What we learnt
We learned how to create a new repository and how to manage it.
We also learnt how to create, categorize, and assign issues. Additionally, how to create branches, add commits using descriptive language, 
and eventually merge them using pull requests.

#Challenges 
Navigating GitHub at the beginning was a bit tricky.
We were confused about how to make more than one commit in the same branch.
Realized that commits had to be made one at a time, separately, to make it meaningful, instead of combining everything in one.

#Fun parts 
Testing the code was fun since we got to see how our weight changed on the moon, Mars, and the International Space Station.
Also, this felt like working on a real project in a software company.

#Future Improvements 
Obviously, this was a very basic idea that could be vastly improved by adding a user interface and including features that allow the user to interact with the tool. Additionally, we would also add more planets.
